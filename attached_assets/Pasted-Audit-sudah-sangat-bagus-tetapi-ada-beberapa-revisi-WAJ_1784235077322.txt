Audit sudah sangat bagus, tetapi ada beberapa revisi WAJIB yang harus diikuti sebelum implementasi. Jangan mulai menulis kode sampai seluruh revisi di bawah ini dipahami.

==========================
REVISI WAJIB
==========================

1. History BoomBox

Jangan membuat migrasi besar history[] menjadi logs.youtube[].

Gunakan history[] sebagai sumber data utama.

BoomBox Logs Viewer cukup memfilter berdasarkan platform:

- YouTube
- TikTok
- Spotify

Dengan begitu semua log lama otomatis tetap terbaca tanpa perlu memindahkan data.

Jika nanti diperlukan logs terpisah, tetap harus backward compatible dengan history lama.

==========================

2. Cache

Jangan memindahkan seluruh cache ke database.

Tetap gunakan:

- Memory Cache
- Metadata Cache

karena jauh lebih cepat.

Database hanya boleh digunakan untuk data cache yang memang perlu dipertahankan setelah restart apabila benar-benar diperlukan.

Prioritaskan performa.

==========================

3. setup.js

Jangan rename.

Jangan redirect.

Jangan ubah command /setup yang sudah ada.

BoomBox mempunyai command sendiri:

/setupboombox

Keduanya harus tetap bisa digunakan.

==========================

4. Role Duration

Jangan menyimpan nama role secara permanen di database.

Bot harus membaca role Discord secara otomatis setiap kali panel dibuka.

Database cukup menyimpan:

Role ID
↓

Durasi Maksimal

Nama role selalu diambil langsung dari Discord.

Jika owner membuat role baru maka otomatis muncul.

Jika role dihapus maka otomatis hilang.

Tidak boleh hardcode.

==========================

==========================

6. Cleaner

Cleaner harus diperluas.

Yang boleh dibersihkan:

- Temporary Files
- Metadata Cache
- Upload Cache
- Download Cache
- Expired Cache
- Orphan Files
- Junk Files
- Attached Assets yang sudah tidak digunakan
- Temporary Logs

Yang TIDAK BOLEH dihapus:

- Database
- Premium
- Limit
- Setup BoomBox
- History
- BoomBox Logs
- Backup
- Config
- URL BoomBox

Cleaner harus aman.

==========================
