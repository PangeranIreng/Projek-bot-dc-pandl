==========================================================
AUDIT
==========================================================

Baca seluruh source BoomBox terlebih dahulu.

Audit seluruh file yang berhubungan dengan:

BoomBox

Queue

Downloader

yt-dlp

Analytzy

HTTP Request

Worker

Concurrency

FFmpeg

Timeout

Retry

Error Handler

Cari seluruh kemungkinan penyebab BoomBox bisa stuck.

JANGAN langsung mengubah kode sebelum audit selesai.

==========================================================
TARGET
==========================================================

BoomBox HARUS tetap berjalan 24/7.

BoomBox TIDAK BOLEH berhenti hanya karena satu provider error.

BoomBox HARUS bisa dipakai banyak user secara bersamaan.

BoomBox HARUS tetap berjalan walaupun Scanner Keylogger sedang melakukan scan.

BoomBox tidak boleh mengganggu Scanner.

Scanner tidak boleh mengganggu BoomBox.

Semua fitur harus berjalan bersamaan.

==========================================================
PROVIDER SYSTEM
==========================================================

Gunakan sistem Multi Provider.

Prioritas:

1. Analytzy (Utama)

↓

2. yt-dlp Local

↓

3. Backup API 1

↓

4. Backup API 2

↓

5. Backup API 3 (jika memang sudah ada di project)

JANGAN hanya bergantung pada satu provider.

==========================================================
FALLBACK
==========================================================

Jika Analytzy gagal:

Timeout

HTTP Error

429

403

Cloudflare

Download gagal

Video info gagal

Network Error

DNS Error

Socket Error

langsung pindah ke provider berikutnya.

JANGAN berhenti.

JANGAN membuat Queue macet.

==========================================================
HEALTH CHECK
==========================================================

Buat sistem Health Check.

Misal:

Analytzy gagal 5 kali berturut-turut.

↓

Status provider menjadi:

OFFLINE

Selama OFFLINE:

BoomBox JANGAN lagi mencoba Analytzy.

Langsung gunakan provider berikutnya.

Setelah beberapa menit (misal 10 menit):

Bot otomatis mencoba Analytzy lagi.

Jika berhasil:

Status menjadi:

ONLINE

Jika gagal lagi:

Tetap OFFLINE.

Tidak perlu restart bot.

Tidak perlu command manual.

==========================================================
QUEUE
==========================================================

Queue harus benar-benar stabil.

Setiap Job memiliki:

Timeout

Retry

Cancel

Cleanup

Finally

Apapun hasil Job:

SUCCESS

FAILED

TIMEOUT

CANCEL

Slot Queue WAJIB dilepas.

Tidak boleh ada Queue Lock.

Tidak boleh ada Queue Stuck.

==========================================================
TIMEOUT
==========================================================

Jangan menggunakan timeout yang membuat Queue berhenti.

Timeout hanya menghentikan Job tersebut.

Bukan seluruh BoomBox.

==========================================================
YT-DLP
==========================================================

Jangan download yt-dlp setiap request.

Jika binary sudah ada:

Langsung gunakan.

Jika binary tidak ada:

Download sekali.

Simpan.

Gunakan kembali.

Jika gagal download:

Gunakan provider berikutnya.

==========================================================
DOWNLOAD
==========================================================

Gunakan kualitas audio yang ringan.

Target:

128 kbps

atau kualitas terendah yang masih layak.

Prioritaskan:

Ukuran kecil

Cepat

Stabil

Bukan kualitas tinggi.
